Online Bookstore Management System

The Online Bookstore Management System is a web-based application built using Spring Boot and Java 17 as part of our Object-Oriented Programming coursework at SLIIT. The aim of this project was to build a complete bookstore platform that allows users to register, explore books, view offers, make payments, and buy stationery or tickets‚Äîall without using a traditional database. Instead of SQL, we used file handling to persist data, offering a clear demonstration of core OOP principles, Java I/O, and MVC architecture in a real-world web application.

üß© Project Structure & Purpose

We followed the MVC (Model-View-Controller) pattern to structure the application into logical, manageable components.

üìÅ Controllers (src/main/java/.../controller)

These files handle user interactions and route requests to the correct services:
  • HomeController - Handles the homepage and navigation.
  • OfferController - Manages promotional offers and discounts.
  • PaymentController - Processes payments and transactions.
  • StationaryController - Displays stationery items.
  • TicketController - Manages event or transport tickets.
  • UserController - Manages user registration and login.

    • üìÅ Models (src/main/java/.../model)

    These files define the structure of data objects used throughout the system:

  • Book, Offer, Payment, Stationary, Ticket, User - Each class represents a real-world entity with relevant attributes.

    • üìÅ Services (src/main/java/.../service)

    These contain the business logic and file-handling methods:

  • Each service (e.g., BookService, UserService) reads from and writes to .txt files stored in the data folder.

    • üìÇ Data Storage

  • /data/ - Stores .txt files for all data (e.g., books.txt, users.txt, offers.txt) acting as a lightweight file-based database.
  • /data/image/ - Stores image files (e.g., book covers, product images).

    • We created a Constants file to manage file paths centrally because Spring Boot blocks access to local file system paths for security reasons. This helps us access images reliably from within the application.

    üñº JSP & Web Interface

    All front-end views are stored in:
  • /webapp/WEB-INF/jsp/
  • These JSP files are rendered by the controllers and dynamically display data from the .txt files.
  • Examples: home.jsp, books.jsp, login.jsp, offers.jsp, etc.
